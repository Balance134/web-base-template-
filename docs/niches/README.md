# Система ниш — design overrides

Эта папка содержит правила для конкретных бизнес-ниш, которые имеют приоритет над дефолтами скилла UI/UX Pro Max.

## Как работает

Каждая ниша = отдельный файл (`hotels.md`, `restaurants.md`, `clinics.md`, и т.д.). Файл содержит правила для color, typography, style, effects, CTA, photos, references, заточенные под эстетику и аудиторию конкретной ниши.

## Когда я начинаю работу над клиентом

1. Я говорю Claude Code: "работаю над <тип бизнеса>" (например "работаю над клиникой эстетической медицины").
2. Claude Code проверяет — есть ли `docs/niches/<niche>.md`?
   - Если ДА — использует правила оттуда поверх скилла.
   - Если НЕТ — активирует `@niche-design-researcher` для создания файла этой ниши.

## Создание новой ниши через @niche-design-researcher

Агент:
1. Через web_search исследует 5-7 топовых брендов в нише.
2. Извлекает закономерности: палитры, шрифты, стиль, эффекты, CTA-паттерны, фото-подход.
3. Заполняет шаблон `_template.md` по структуре существующих файлов.
4. Показывает мне черновик для подтверждения.
5. После моего ОК — сохраняет как `docs/niches/<niche>.md`.

Это ~5-10 минут на новую нишу один раз. Дальше эта ниша используется автоматически для всех будущих клиентов.

## Приоритет

Когда правила в `docs/niches/<niche>.md` противоречат дефолтам скилла UI/UX Pro Max — приоритет у файла ниши.
Когда правила в файле ниши противоречат правилам в корневом `CLAUDE.md` (например, запрет stock-фото) — приоритет у `CLAUDE.md`.

## Порядок применения (от высокого к низкому)

1. `CLAUDE.md` (корневые правила проекта)
2. `docs/niches/<niche>.md` (правила ниши)
3. UI/UX Pro Max skill defaults
