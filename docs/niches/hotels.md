# Бутик-отели

> Правила для проектов в нише: бутик-отели и премиальные гостиницы. Аудитория — туристы 35-60 из СНГ и Европы, для региона Узбекистан также путешественники, ищущие аутентичный опыт.

## Целевая аудитория

- Возраст: 35-60 лет, обеспеченные путешественники
- Контекст использования: преимущественно mobile (в роуминге, со средним интернетом), часть — desktop при планировании поездки
- Что важно для аудитории: трастовость, эстетика как обещание опыта, читаемость на мобильном, скорость загрузки в роуминге, реальные фото номеров и территории

## Color palette

Четыре допустимые палитры — выбор зависит от позиционирования и локации отеля:

**Classic luxury (универсальная, городские/курортные отели премиум-класса):**
- Background: `#FAF8F4` (off-white кремовый)
- Text: `#1A2A44` (глубокий navy)
- Accent: `#C9A961` (приглушённое золото)
- Muted: `#8B8680`

**Warm Bukhara (для отелей в Бухаре/Самарканде, ремесленная/историческая эстетика):**
- Background: `#F5EFE3` (тёплый песочный)
- Text: `#3A2418` (тёмно-коричневый)
- Terracotta: `#B4623A`
- Ochre: `#D4A24C`

**Cool mountain (для горных/природных локаций — Чимган, эко-ретриты):**
- Background: `#F4F2EC`
- Text: `#1F2E26` (тёмно-зелёный)
- Forest: `#3F5D4A`
- Stone: `#A8A39B`

**Urban Tashkent (для современных городских бутиков):**
- Background: `#F2F0EE`
- Text: `#1C1C1C` (почти чёрный)
- Dusty pink: `#C89A9A`
- Gold: `#B8945C`

- НЕ использовать: чистый `#FFFFFF` для фона (слишком клинично, читается как медицина/SaaS), яркие saturated цвета (#FF0000, #00FF00 и подобные неоновые), trendy gradient backgrounds, пурпурно-фиолетовые SaaS-палитры
- Обоснование: премиальные отели задают спокойную, тёплую, чуть приглушённую палитру — это считывается аудиторией как «дорого и со вкусом». Референс — Aman, Six Senses, Belmond, Rosewood.

## Typography

- Heading: **Playfair Display**, **Cormorant Garamond**, **DM Serif Display**, **Fraunces** — serif передают традицию, мастерство, культурный пласт; критично для отельной ниши
- Body: **Inter**, **Manrope**, **Geist** — современный sans для читаемости длинных описаний номеров и условий бронирования
- НЕ использовать: Inter для заголовков (дефолт скилла — для отелей выглядит как SaaS-лендинг, разрушает премиальность), декоративные шрифты типа Comic Sans / Lobster / Pacifico, monospace для тела (читается как технический док)

## Style

- Использовать: **Minimalism** (воздух, крупные фото, мало текста), **Editorial** (типографические композиции, цитаты, story-driven layout), **Immersive Photography** (full-bleed фото территории/номеров с текстом поверх) — почему: эти стили совпадают с тем, как делают Aman, Six Senses, Belmond
- НЕ использовать: **Liquid Glass** / glassmorphism (читается как tech/SaaS, не отельная эстетика), **Brutalism** (агрессивно для аудитории 35-60), **Bento-grid в hero** (фрагментирует первое впечатление; bento допустим в секциях ниже — например, для услуг)

## Hero pattern

- Layout: full-bleed фото 80-100vh высоты, тёмный градиент-оверлей `linear-gradient(180deg, rgba(0,0,0,0.1) 0%, rgba(0,0,0,0.5) 100%)` для контраста текста
- Primary visual: реальное фото территории/фасада/знакового интерьера отеля (с Booking-страницы клиента или с фотосессии). Если фото ещё нет — typographic hero на градиентном фоне (см. Photos → fallback)
- CTA placement: один primary CTA по центру или в правой нижней четверти, под заголовком; secondary link типа «Посмотреть номера» — второй, без фона
- Heading: serif 64-88px на desktop, 36-48px на mobile

## CTA

- Текст: **«Забронировать»** / **«Book your stay»** / **«Bron qilish»** (RU/EN/UZ). Допустимо: «Проверить даты», «Check availability». НЕ использовать: «Купить», «Получить скидку», агрессивные формулировки
- Расположение: primary в hero, sticky на mobile внизу экрана, повтор в конце каждой ключевой секции (номера, ресторан, локация)
- Цвет: accent цвет палитры (золото / terracotta / forest) на тёмном или светлом фоне с достаточным контрастом (WCAG AA min). Минимальный hit area 44×44px (Apple HIG)

## Photos

- Допустимые источники:
  - Реальные фото клиента в `/public/images/` с осмысленными именами (`courtyard-evening.jpg`, `room-deluxe-01.jpg`, `restaurant-terrace.jpg`)
  - Фото с Booking-страницы клиента (это РЕАЛЬНЫЕ фото отеля)
  - Профессиональная фотосессия, заказанная отелем
- Запрещено: Unsplash, Pexels, Pixabay, Picsum, любые stock-фото — даже как временные плейсхолдеры. Причина: ИИ не «видит» картинки и опирается на метаданные/теги, которые врут — на тестах поиск по «Samarkand»/«Registan»/«boutique hotel» дважды отдал не ту страну (Шефшауэн в Марокко, Берлин). Поставить чужую страну на главную страницу отеля — недопустимо.
- Резервный режим если нет фото: **typographic hero** — крупный serif заголовок на градиентном фоне в палитре отеля, без фото вообще. Часто сильнее, чем «лишь бы какая картинка». Подробности — `docs/section-by-section-workflow.md`.

## Effects (animations)

- Допустимо:
  - Fade-in на scroll (Intersection Observer, `opacity 0 → 1`, 400-600ms)
  - Лёгкий parallax на hero-фото (translateY до 10-15% от scroll, не больше)
  - Плавные переходы 200-300ms для hover/focus состояний
  - Stagger-анимация для появления списка карточек (delay 50-80ms между элементами)
- НЕ использовать: morphing-эффекты, heavy backdrop-blur (>20px), автоплей фонового видео (особенно с звуком), chromatic aberration, scroll-jacking, parallax с мультислоями, тяжёлые WebGL-сцены
- Причина: основная аудитория смотрит сайт с мобильного, часто в роуминге со слабым сигналом. Тяжёлые эффекты убивают LCP и съедают батарею — это считывается как «непрофессионально», обратный эффект от того, что задумывалось.

## Notes

**Узбекистан-специфика:**
- **Языки персонала (`amenities.languages_spoken` в `hotel.json`):** `узбекский` добавляется по умолчанию — это база региона (как русский в России). Русский и английский — только если подтверждены на Booking. Другие языки — только если явно указаны в источниках.
- **Три языка UI обязательно:** RU + EN + UZ. В этом Next.js проекте — через `next-intl` + `messages/uz.json` (создаём сразу, даже если наполнение пойдёт позже). Включаем `uz` в `i18n/routing.ts` наравне с `ru`/`en`. Для статического `hotel-pitch-template` те же три языка реализуются через JS-переключатель в HTML — реализация по запросу при сборке конкретного превью.
- **Локальная эстетика для Бухары/Самарканда:** допустимы тонкие отсылки к ремесленной традиции — золотые орнаменты Регистана/Шахи-Зинды как декоративный элемент в малых дозах (НЕ как фон, НЕ кричаще), тёплая земельная палитра Бухары. Не уходить в туристический китч.
- **Performance budget:** LCP < 2.5s на mobile в 3G, JS bundle < 200KB gzipped. Это жёстче дефолтных рекомендаций — потому что аудитория в роуминге.
- **Фото с Booking** — основной источник на старте проекта, до собственной фотосессии клиента. Парсер — `scripts/` (см. план проекта).
